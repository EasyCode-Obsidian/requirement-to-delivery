# Task Todo

> Adapt this template to the approved task before execution begins.

## Part 1: Requirement and scope
### 1.1 Clarify the implementation target
- [ ] Task: Rewrite the approved request into a concrete implementation objective
- Goal: Make the execution target explicit and testable
- Done when: Scope, constraints, deliverables, and success criteria are written clearly
- Deliverables: Approved requirement summary
- Notes: Record assumptions or unresolved edge cases

### 1.2 Confirm constraints and dependencies
- [ ] Task: List technical, product, workflow, and environmental constraints
- Goal: Prevent execution drift and hidden blockers
- Done when: Constraints and dependencies are documented
- Deliverables: Constraint and dependency list
- Notes: Include blockers, required tools, or approvals

## Part 2: Research and planning
### 2.1 Review relevant workspace context
- [ ] Task: Inspect existing files, code, or documents related to the task
- Goal: Reuse context before inventing new structure
- Done when: Relevant local context is summarized
- Deliverables: Context summary
- Notes: Reference important paths or components

### 2.2 Identify relevant skills and tools
- [ ] Task: Select the skills, MCP tools, and local utilities that can help with execution
- Goal: Use the strongest available capabilities at the right time
- Done when: Relevant capabilities are identified with a reason for use
- Deliverables: Capability shortlist
- Notes: Avoid unrelated tools or deep reads

## Part 3: Implementation
### 3.1 Execute the first implementation slice
- [ ] Task: Complete the first concrete work item from the approved plan
- Goal: Convert the approved draft into visible progress
- Done when: The slice is implemented and validated at an appropriate level
- Deliverables: Updated files, code, or artifacts
- Notes: Capture validation steps and any follow-up work

### 3.2 Complete remaining implementation slices
- [ ] Task: Finish the remaining work defined by the approved plan
- Goal: Deliver the requested outcome without scope drift
- Done when: All planned slices are completed or intentionally deferred
- Deliverables: Final implemented artifacts
- Notes: Add new subtasks if the plan needs refinement

## Part 4: Verification and completion
### 4.1 Verify outcomes
- [ ] Task: Run the appropriate checks for the completed work
- Goal: Ensure the work is truly complete before marking it done
- Done when: Validation results support completion
- Deliverables: Test, review, or verification summary
- Notes: Keep the task unchecked if validation is blocked or incomplete

### 4.2 Finalize and report
- [ ] Task: Summarize what was done and update completion status honestly
- Goal: Leave a clear audit trail for the user and future work
- Done when: All completed subtasks are marked `[x]` and the final summary is ready
- Deliverables: Final status summary
- Notes: Record deferred items, known limitations, or next steps
