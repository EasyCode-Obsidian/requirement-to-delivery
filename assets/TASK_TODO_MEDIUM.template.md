# Task Todo - Medium

> Use this file as the reviewable and communication-friendly plan.
> It should be easy for the user to scan and understand overall scope and progress.

## Part 1: Requirement and scope
### 1.1 Clarify the implementation target
- [ ] Task: Rewrite the approved request into a concrete implementation objective
- Goal: Make the execution target explicit and testable
- Done when: Scope, constraints, deliverables, and success criteria are written clearly
- Deliverables: Approved requirement summary
- Notes: Record assumptions, unresolved edge cases, or scope boundaries

### 1.2 Confirm constraints and dependencies
- [ ] Task: Summarize the main constraints, dependencies, and external conditions
- Goal: Prevent execution drift and hidden blockers
- Done when: Constraints and dependencies are documented at a planning level
- Deliverables: Constraint and dependency summary
- Notes: Include blockers, required tools, approvals, or high-level risks

## Part 2: Research and planning
### 2.1 Review relevant context
- [ ] Task: Identify and summarize the most relevant existing context for the task
- Goal: Reuse known information before creating new structure
- Done when: Relevant local or external context is summarized at a high level
- Deliverables: Context summary
- Notes: Reference important files, sources, systems, or prior decisions

### 2.2 Establish the execution plan
- [ ] Task: Define the main workstreams and execution sequence
- Goal: Create a stable plan that can be reviewed and refined
- Done when: Major workstreams are identified in a logical order
- Deliverables: High-level execution plan
- Notes: Keep the plan concise and user-reviewable

## Part 3: Execution
### 3.1 Complete the first major workstream
- [ ] Task: Finish the first major portion of the approved plan
- Goal: Turn the plan into visible progress
- Done when: The first workstream is completed at a meaningful level
- Deliverables: Updated artifacts, outputs, or decisions
- Notes: Link major outcomes to the fine-grained todo

### 3.2 Complete remaining major workstreams
- [ ] Task: Finish the remaining approved workstreams
- Goal: Deliver the requested outcome without scope drift
- Done when: All planned workstreams are completed or intentionally deferred
- Deliverables: Final implemented artifacts
- Notes: Record deferred items or agreed scope changes

## Part 4: Verification and completion
### 4.1 Verify outcomes
- [ ] Task: Confirm that the completed work satisfies the approved plan
- Goal: Ensure the work is actually ready to report as done
- Done when: Validation results support completion
- Deliverables: Verification summary
- Notes: Keep unchecked if validation is blocked or incomplete

### 4.2 Finalize and report
- [ ] Task: Summarize outcomes and update completion status honestly
- Goal: Leave a clear high-level audit trail
- Done when: Completed items are marked `[x]` and the final summary is ready
- Deliverables: Final status summary
- Notes: Record next steps, limitations, or follow-up work
